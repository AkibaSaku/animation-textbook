RPC · HTTP/2 · Protobuf

型付きの直通電話gRPC

遠くのサーバーの関数を、まるで手元の関数のように呼びたい。gRPC はそのための型付きの直通電話です。先に .proto で「どんな呼び出しがあり、引数と戻り値の型は何か」を契約し、そこからクライアントとサーバー両方のコードを自動生成。通信はHTTP/2の上を、Protobufでバイナリ圧縮したメッセージが流れます。JSONより小さく速く、双方向の流しっぱなし通話もできます。

01契約書（.proto）から両端を生成

サービスとメッセージの型を1枚に書く。これがSOAPのWSDLに当たる「型の真実」です。

service PriceService { rpc GetPrice(ItemReq) returns (PriceResp); # 1往復 rpc WatchPrice(ItemReq) returns (stream PriceResp); # 流しっぱなし } message ItemReq { string item = 1; } message PriceResp { int32 price = 1; }

024つの通話モード

HTTP/2の双方向ストリームのおかげで、1往復だけでなく「流しっぱなし」も選べます。モードを選んで再生してください。

モード
モードを選んで「通話」を押してください

Protobufのキモ： 送られるのは {"item":"apple"} のような文字ではなく、フィールド番号で詰めた短いバイナリ（紫の粒）。両端が同じ .proto を知っているからキー名を送る必要がなく、軽くて速い。型違いは生成段階で弾かれます。

03REST/JSON との使い分け

gRPC が向く

社内のマイクロサービス間、低遅延・高頻度の通信、ストリーミング。型が厳密で速い。

REST/JSON が向く

ブラウザから直接叩く公開API、人が目で読みたい場面。どこでも繋がり手軽。

つながり： SOAP/WSDLの「契約してコード生成」という発想を、HTTP/2＋バイナリ＋ストリームで現代化したのがgRPC。ブラウザから使うときは gRPC-Web という橋渡しを挟みます。
