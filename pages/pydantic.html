データの関所Pydantic

外から届くデータは形がバラバラで信用できません。Pydanticは「こういう型のはず」という設計図を関所に立て、届いたデータを検査します。直せるものは正しい型に変換し、ダメなものは却下。

01関所に立てる設計図

「ユーザーとは id(整数)・name(文字列)・age(整数) を持つもの」と宣言します。

from pydantic import BaseModel class User(BaseModel): id: int name: str age: int # 0以上のはず

02データを通してみる

ボタンで色々なデータを関所に送ります。通過・変換・却下のどれになるか見てください。

// ボタンを押すとデータが関所に向かいます
通過 変換して通す 却下

キモ："42"という文字列でidを送っても整数42に自動変換。"abc"は整数にできないので却下。

03なぜ嬉しいのか

関所がない世界

使う場所ごとに型確認だらけ。書き忘れた所でいつか落ちる。

関所がある世界

入口で一度検査済み。以降は型が保証され安心して書ける。
