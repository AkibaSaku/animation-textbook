Browser Security · Same-origin

他オリジンへの出入国審査CORS

ブラウザには同一オリジン原則があり、app.example のページが api.other.com を勝手に読むのを止めます（悪意あるサイトの防止）。でも正当に他所のAPIを使いたいことも多い。CORS は「相手サーバーが明示的に許可していれば通す」ための出入国審査。鍵を握るのはブラウザで、サーバーの返すヘッダを見て、JSへ結果を渡すかどうかを決めます。

01プリフライトという事前審査

書き込み系など「単純でない」リクエストの前に、ブラウザは OPTIONS で先に許可を尋ねる。サーバーの許可設定を切り替えて試してください。

サーバーの許可
許可設定を選んで「リクエスト」を押してください

よくある誤解： CORSで拒否されてもサーバーには届いていることが多い。ブロックしているのはブラウザがJSへ応答を渡す段階。だから「サーバー側のヘッダ設定」で直すもので、フロント側のコードでは回避できません。

02許可を表すヘッダ

サーバーは応答にこれらを付けて「君のオリジンはOK」と告げます。

Access-Control-Allow-Origin: https://app.example Access-Control-Allow-Methods: GET, POST, PUT Access-Control-Allow-Headers: Content-Type, Authorization Access-Control-Max-Age: 86400 # 審査結果を1日キャッシュ

03単純リクエストとそうでないもの

単純リクエスト

GET/POSTで素朴なヘッダのみ等。プリフライト無しで送られ、応答ヘッダだけで判定。

プリフライト必要

PUT/DELETE、Authorization や独自ヘッダ付きなど。先に OPTIONS で事前確認。

オリジンとは： スキーム + ホスト + ポート の3点セット。1つでも違えば「別オリジン」でCORSの対象。httpsとhttp、:80と:8080 も別物です。

04なぜ同一オリジン原則が要るのか

CORSを理解する鍵は「ブラウザはリクエストにクッキーを自動で付ける」点です。これが無防備だと、こんな攻撃が成立してしまいます。

# あなたが bank.example にログイン済み（クッキー保持） # その状態で罠サイト evil.example を開くと… fetch("https://bank.example/transfer", { method: "POST", credentials: "include" # あなたのクッキーが自動で付く })

罠サイトのJSが、あなたのログイン状態を借用して銀行APIを叩けてしまう。これを防ぐのが同一オリジン原則と、その緩和を厳格に管理するCORSです。

CORSが守るもの

他オリジンのJSが応答の中身を読むこと。許可ヘッダが無ければ、ログイン状態で取得した個人情報をJSが盗み見られない。

CORSが守らないもの

リクエストが届くこと自体は止めない。単純リクエストはプリフライト無しで送信され、副作用（書き込み）が起きうる。

つまりCORSは「読み取り防御」： 応答の漏洩は防ぐが、リクエスト送信自体は防がない。状態を変える操作（送金・削除など）の保護はCSRFトークンや SameSite クッキーなど別の仕組みが担います。CORSはCSRF対策ではありません。

05設定でやりがちな危険パターン

CORSは緩めるほど危険。よくある事故を3つ挙げます。

# ✗ 危険1: 何でも許可（個人情報APIでこれは事故） Access-Control-Allow-Origin: * # ✗ 危険2: 「*」とクレデンシャルの併用（仕様上そもそも不可） Access-Control-Allow-Origin: * Access-Control-Allow-Credentials: true # ブラウザが拒否 # ✗ 危険3: Originをそのまま反射（=実質what全許可） Allow-Origin: <リクエストのOriginをそのままコピー>

なぜ「*」が危険か

認証付きの個人データを返すAPIで全許可すると、どんなサイトからでも応答を読める。許可は信頼するオリジンを明示列挙が原則。

Origin反射の罠

来たOriginを検証せず鏡のように返すと、攻撃者のオリジンも通る。許可リストとの照合を必ず挟む。

実務の指針： ①許可オリジンは明示リストで照合 ②クレデンシャルを使うなら*不可・具体的オリジン必須 ③許可メソッド/ヘッダは必要最小限 ④状態変更操作はSameSite＋CSRF対策を別途。CORSは入口の鍵であって、金庫の鍵ではないと覚えておくと安全です。
