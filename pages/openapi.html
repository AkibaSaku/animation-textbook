APIのメニュー表と試食カウンターOpenAPI & Swagger

APIは「何が頼めるか」を知らないと使えません。OpenAPI はそのメニュー表（仕様書）。どんな料理（エンドポイント）があり、何を注文すれば何が出てくるかを機械可読な形で書いた1枚の文書です。Swagger UI はそのメニューからその場で試食できるカウンター。仕様書を読み込んで、ブラウザ上でAPIを叩けるUIを自動生成します。

011枚のメニュー表（OpenAPI仕様）

YAML/JSONで「パス・メソッド・引数・レスポンスの形」を宣言します。これが唯一の真実の источник。

openapi: 3.0.0 paths: /users/{id}: get: summary: ユーザーを1件取得 parameters: - name: id # パスの引数 schema: { type: integer } responses: 200: { id, name, email } # 返る形

02試食カウンター（Swagger UI）で叩く

仕様書から自動生成されたUI。エンドポイントを開いて Try it out を押すと、本物のリクエストが飛びます（ここは模擬です）。

Swagger UI · petstore-demo
GET/users/{id}ユーザーを1件取得
// レスポンスがここに表示されます
POST/usersユーザーを新規作成
// レスポンスがここに表示されます

ここが効く： 仕様書を更新するだけでUIも自動で更新される。ドキュメントと実物がズレない。引数の型もUI側が知っているので、入力チェックや例の表示もタダで付いてきます。
